# YouthPlayer 新手使用说明（完整版本）

本文档面向第一次接触本系统的老师/值班同学/运维人员，按“从零到可用”的顺序编写。

## 1. 系统是什么

YouthPlayer 是一个基于 Web 的视频播放与排期系统，主要能力：

- 定时播放（一次性任务、每周循环任务）
- 多视频播放列表（支持批量导入、拖拽排序）
- 本地/NAS/网络 URL/Nextcloud 视频源
- 多屏播放（指定播放屏幕）
- 窗口控制（全屏或自定义窗口位置/大小）
- 空闲屏保（无计划时显示）
- 画面监控截图
- 多用户和权限管理
- 操作与设置审计日志

播放后端说明：

- 当前版本以 Electron 为主播放引擎（本地、NAS、直播页、直链视频都可走 Electron）。
- VLC 相关逻辑仍保留，但常规配置可优先使用 Electron。

## 2. 运行环境要求（Windows）

- Windows 7/10/11（推荐 10 或 11）
- Python 3.9+（`setup_env_windows.bat` 里按 3.9+ 设计）
- Node.js + npm（用于安装 Electron 依赖，若只用现成环境可跳过）
- 网络可达你的 NAS/Nextcloud 地址

项目默认端口：

- Web 管理界面：`5000`
- Electron 控制端口：从 `18870` 起自动分配

## 3. 首次安装（推荐脚本方式）

在项目根目录执行：

```bat
setup_env_windows.bat
```

该脚本会做这些事：

- 创建并激活 `.venv` 虚拟环境
- 安装 Python 依赖（`requirements.txt`）
- 尝试执行 `npm install`（安装本地 Electron 相关依赖）

安装完成后启动：

```bat
start_windows.bat
```

或手动方式：

```bat
.venv\Scripts\activate
python main.py
```

## 4. 登录与初始账号

打开浏览器访问：

- `http://127.0.0.1:5000/login`

默认管理员账号：

- 用户名：`admin`
- 密码：`admin123`

首次登录后请立刻在“系统设置”里修改管理员密码。

## 5. 页面结构（导航）

登录后顶部导航通常包含：

- 控制台
- 播放计划
- 画面监控
- 用户管理（需权限）
- 系统设置（需权限）

具体看到哪些菜单，取决于账号权限。

## 6. 快速上手（10 分钟完成）

建议新手按这个顺序：

1. 进入“系统设置”，确认播放模式、屏保、Nextcloud（如需）。
2. 进入“播放计划”，新增一个测试计划。
3. 在“计划列表”点“立即播放”验证。
4. 到“画面监控”确认有截图刷新。
5. 回“系统设置”修改管理员密码，创建普通账号并分配权限。

## 7. 播放计划：核心概念

### 7.1 时间模式

- 一次性任务：按完整日期时间运行（开始到结束）。
- 每周循环：按星期几 + 当天时间段运行。

注意：

- 每周循环必须勾选至少 1 个星期。
- 每周循环仅支持“同一天内”时间段（结束时间必须晚于开始时间，不能跨天）。

### 7.2 内容来源

- 本地视频：如 `D:\video\a.mp4`
- NAS 路径：如 `\\nas\share\a.mp4`
- 网络地址：HTTP/HTTPS 直链或直播页 URL
- Nextcloud：通过文件浏览器选取云端文件（会生成可播放链接）

### 7.3 播放列表与循环

你现在只需要填写“播放列表（一行一个，单视频也填这里）”。

- 第一行会作为默认播放项。
- 可“批量导入视频”。
- 支持拖拽排序。

循环模式：

- `按列表播放一次`：列表播完停止
- `列表循环播放`：列表轮播循环
- `单条循环`：只循环当前条目

循环次数：

- `0` 表示无限循环
- `>0` 表示限制循环轮数

### 7.4 屏幕与窗口

- 播放屏幕：选择显示器编号。
- 窗口模式：
  - 全屏：自动铺满目标屏幕
  - 自定义：可填 `X/Y/宽/高`
- 可视化拖拽框：可直接拖动位置和右下角缩放，系统会自动回填坐标。

## 8. 新建播放计划（一步一步）

进入“播放计划 -> 新建计划”，按下面填：

1. 计划名称：如“午间宣传片”
2. 开始/结束时间：先做一个“当前时间+1分钟”的测试
3. 是否每周循环：测试时可不勾
4. 内容类型：本地/NAS/网络直播页
5. 播放列表：每行一个路径或 URL
6. 循环模式：测试建议“按列表播放一次”
7. 播放屏幕：选目标屏
8. 窗口模式：先用全屏
9. 点“保存计划”

保存后可在“计划列表”中：

- 立即播放
- 启用/停用
- 删除
- 展开后编辑全部参数

## 9. 控制台与手动控制

控制台提供：

- 开始
- 停止
- 暂停
- 恢复
- 网页注入播放/全屏（用于某些网页视频场景）

建议：

- 日常优先通过“计划”管理，手动控制用于临时介入。

## 10. 画面监控

“画面监控”页面可查看最近截图，并支持“手动截图”。

截图频率可在系统设置调整：

- `监控截图频率（秒）`（范围 2~3600）

## 11. 屏保设置（空闲时显示）

在“系统设置”中可配置：

- 屏保图片（jpg/jpeg/png/bmp/webp）
- 屏保显示屏幕
- 屏保窗口模式（全屏/自定义）
- 屏保窗口 X/Y/宽/高

行为说明：

- 当没有活动计划且未手动抑制时，系统会尝试显示屏保页面。

## 12. Nextcloud 配置与使用

### 12.1 配置项

在“系统设置”填写：

- 启用 Nextcloud WebDAV 资源浏览
- Nextcloud 地址（如 `https://cloud.example.com`）
- 用户名
- 应用密码（推荐，不要用主密码）
- 根目录（如 `/` 或 `/videos`）
- 跳过 SSL 证书校验（仅内网自签证书场景使用）

### 12.2 连接测试

点“测试连接”：

- 成功会提示连接正常。
- 失败会显示具体错误（认证失败、HTTP 错误等）。

### 12.3 云端预览

点“预览目录”可浏览云端文件夹结构，选择视频加入播放列表。

### 12.4 播放机制（当前版本）

为提高稳定性，系统会在后端下载 Nextcloud 文件到本地缓存后再交给 Electron 播放。

缓存目录：

- `runtime/nextcloud_cache`

## 13. 缓存管理

系统设置页提供 Nextcloud 缓存统计：

- 文件数量
- 总大小
- 最早/最新文件时间

可点击“Clear Cache”一键清理缓存。

建议：

- 磁盘紧张时清理
- 更新同名资源后清理一次，避免旧缓存干扰

## 14. 用户与权限

管理员可在“用户管理”：

- 新增账号
- 启用/停用账号
- 删除账号
- 为普通账号分配细粒度权限

权限项（代码）：

- `dashboard.view`：查看控制台
- `playback.control`：控制播放
- `schedule.view`：查看播放计划
- `schedule.manage`：管理播放计划
- `monitor.view`：查看画面监控
- `monitor.capture`：手动截图
- `users.manage`：用户管理
- `settings.manage`：系统设置
- `files.browse`：文件浏览与导入

## 15. 系统设置与审计

系统设置包含：

- 全部内容走 Electron
- 监控截图频率
- Nextcloud 参数
- 屏保参数
- 管理员密码修改

审计：

- 设置变更审计日志（谁在何时改了什么）
- 操作审计日志（播放、删除、清缓存等）

- 主数据库：`playback_system.db`

## 17. 常见问题排查（新手友好版）

### 17.1 登录失败

- 确认地址是否是 `http://127.0.0.1:5000/login`
- 确认大小写
- 若忘记密码，需数据库层重置管理员密码

### 17.2 视频播放后不自动切换或不循环

- 检查计划的“循环模式”和“循环次数”
- 循环次数不是 0 时，达到次数会停止
- 看“控制台/状态”与日志确认是否有播放错误

### 17.3 Nextcloud 报证书错误

错误示例：`CERTIFICATE_VERIFY_FAILED`（自签证书）

处理：

- 内网自签场景可勾选“跳过 SSL 证书校验”
- 公网环境不建议关闭证书校验

### 17.4 Nextcloud 缓存下载 404

常见原因：

- 根目录配置不正确
- 文件已移动或路径拼接错误

处理：

- 在“预览目录”中重新定位文件
- 检查 `nextcloud_root` 是否正确

### 17.5 播放窗口灰屏/黑屏

优先检查：

- URL 是否可直接访问
- 是否已完成缓存下载
- 文件编码/容器是否受支持
- Electron 相关报错是否出现在 `runtime/electron_trace.log` 与控制台日志

### 17.6 `Failed to set path`（Electron 主进程报错）

这是 Electron 路径初始化兼容性问题。当前版本已做保护性处理，若仍出现：

- 删除旧进程并重启服务
- 检查当前用户目录权限（尤其 `AppData/Roaming/Electron`）

### 17.7 “Unable to move cache / 拒绝访问”

通常是缓存目录权限或被占用：

- 关闭所有 Electron 相关进程
- 检查目录权限
- 重启后再试

## 18. 备份与恢复建议

最少备份这几项：

- `playback_system.db`
- `runtime/`（可按需，仅建议备份 monitor 图和必要缓存）
- `config.py`（若有本地改动）
- 自定义静态资源（如屏保图）

恢复时优先恢复数据库，再启动服务验证计划和账号。

## 19. 安全建议（务必执行）

- 首次登录立即修改 `admin` 密码
- 普通账号按最小权限分配
- Nextcloud 使用“应用密码”，不要使用主账号密码
- 非必要不要开启“跳过 SSL 证书校验”
- 定期查看审计日志与操作日志

## 20. 运维建议（长期稳定）

- 每周检查一次日志中 ERROR/WARNING
- 每月清理一次 Nextcloud 缓存（按容量）
- 变更前先做小范围测试计划
- 大版本更新前备份数据库

## 21. 给新手值班同学的最短流程卡

1. 登录系统（别忘记账号权限）。
2. 到“播放计划”看今天计划是否启用。
3. 有问题先点“立即播放”验证素材。
4. 到“画面监控”确认实际画面。
5. 若云端素材异常，去“系统设置 -> Nextcloud 测试连接/预览目录”。
6. 还不行就看 `playback_system.log`，把错误时间点发给运维。
