# 校园电视播放系统架构设计

## 1. 系统架构

### 1.1 整体架构
- **播放模块**：负责视频播放，支持本地、NAS和网络直播
- **控制模块**：管理播放时间表，控制播放状态
- **安全模块**：保证播放窗口不受干扰，实现看门狗功能
- **Web管理界面**：远程监控和控制播放系统
- **数据存储**：存储时间表、用户信息等数据

### 1.2 技术选型
- **播放器**：VLC (libvlc) - 支持多种格式，跨平台
- **Web框架**：Flask - 轻量级，适合小型应用
- **数据库**：SQLite - 无需额外安装，适合本地应用
- **定时任务**：APScheduler - 用于时间表管理
- **屏幕管理**：pywin32 - Windows系统屏幕控制
- **网络请求**：requests - 用于网络直播和API调用
- **认证**：Flask-Login - 用户认证

## 2. 核心功能模块

### 2.1 播放模块
- 支持本地视频播放（MP4, AVI, MKV等格式）
- 支持NAS视频播放（通过网络路径访问）
- 支持网络直播（通过URL访问）
- 支持网页直播的打开、关闭、全屏操作
- 避免触发直播页面的防脚本机制

### 2.2 控制模块
- 时间表管理：设置播放时间、内容、顺序
- 播放控制：开始、暂停、停止、切换内容
- 多屏幕管理：控制播放屏幕和播放屏幕大小
- Windows 7兼容性保证

### 2.3 安全模块
- 播放窗口配置：指定播放窗口，防止被其他应用干扰
- 看门狗程序：监控播放状态，防止程序意外终止
- 自动恢复机制：当程序异常时自动重启

### 2.4 Web管理界面
- 远程监控：查看当前播放状态
- 远程控制：控制播放内容和状态
- 时间表管理：在线编辑播放时间表
- 多账户管理：支持团队协作

## 3. 系统流程

1. **启动流程**：
   - 系统启动时加载配置文件
   - 初始化播放模块和控制模块
   - 启动Web服务
   - 启动看门狗程序

2. **播放流程**：
   - 根据时间表触发播放任务
   - 加载对应内容（本地、NAS或网络直播）
   - 在指定屏幕上播放
   - 监控播放状态

3. **Web管理流程**：
   - 用户登录Web界面
   - 查看当前播放状态
   - 编辑时间表或控制播放
   - 保存配置并更新系统

## 5. 技术实现要点

### 5.1 多格式支持
- 使用VLC作为底层播放器，支持多种视频格式
- 对于网络直播，使用嵌入式浏览器或直接流播放

### 5.2 多屏幕支持
- 使用pywin32库获取和控制Windows屏幕
- 支持多显示器配置

### 5.3 防脚本机制规避
- 使用模拟用户操作的方式控制网页直播
- 避免频繁的自动化操作触发防脚本机制

### 5.4 看门狗实现
- 定期检查播放进程状态
- 当发现异常时自动重启播放服务

### 5.5 远程管理
- 基于Flask的Web服务
- 实时状态更新
- 权限控制

## 6. 系统部署

### 6.1 硬件要求
- Windows 7操作系统

### 6.2 软件依赖
- Python 3.8+
- VLC播放器
- 必要的Python库（Flask, APScheduler, pywin32等）

### 6.3 安装步骤
1. 安装Python 3.8+
2. 安装VLC播放器
3. 安装Python依赖包
4. 配置系统参数
5. 启动服务
